Q 1. 
                   function counter(){
                    var counter = 0;
                
                    function IncreaseCounter() {
                        return counter += 1;
                    };
                
                    return IncreaseCounter;
                }
                
                var counter = counter();
                alert(counter());
                alert(counter());
                alert(counter());
                alert(counter());

4 time alert is shown with counter value 1,2,3,4 respectivly

EXPLANATION - when counter() function is being called then it return the IncreaseCounter() function and a counter variable (local variable) because of closure and when the couter() (global first class function) function is being called inside the alert it run IncreaseCounter() function that has use a counter variable which is not in the IncreaseCounter's local memory so it starts finding in the lexical scope and due to closure it uses the counter variable (local variable of counter() function) and increase its value and also update the counter variable. then again alert(counter()); is being call which increase the value of counter (local variable of closure) whos value is increased by 1 because of the previous alert(counter()); so in increase the value of counter variable.

Q 2. 
                let count = 0;
                (function () {
                if (count === 0) {
                    let count = 1;
                    console.log(count); // What is logged?
                }
                console.log(count); // What is logged?
                })();

1
0

EXPLANATION - There is count in the if condition, so to find the the value of count is search in the lexical scope where count =0 (a global variable) so, condition is satisfied now 1 st console .log(count) will print with the value of 1 because of the local variable in the local memory (let count=1) then another console.log(count) will search for count and it prints count 0.
